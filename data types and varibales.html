Variables and Data Types in C

1. What is a Variable?

A variable is a named storage location in memory that holds data during program execution. Each variable has a name, a data type, and a memory location.

Syntax:

data_type variable_name = value;
Example: 
int age = 20;
float height = 5.9;
char grade = 'A';
Notes:
- Variables are case-sensitive.
- Cannot use C keywords as variable names.
- Can start with a letter or underscore and contain letters, digits, or underscores.

2. Data Types in C

C is a statically typed language; every variable must have a data type defined at compile time.

2.1 Primary Data Types

Data Type Description Size (typical) Example
int Integer numbers 2–4 bytes int a = 10;
float Single-precision floating-point 4 bytes float f = 5.5;
double Double-precision floating-point 8 bytes double d = 3.14159;
char Single character 1 byte char c = 'A';
void No data / no return value 0 void func();

2.2 Derived Data Types

  • Arrays – Collection of same-type elements
  • Pointers – Store memory address of a variable
  • Structures – Group of different data types
  • Unions – Store different data types in same memory

2.3 User-Defined Data Types

  • typedef – Define custom type names
  • enum – Set of named integer constants
typedef unsigned int uint;
enum Days {MON, TUE, WED};

2.4 Modifiers

Modifier Description Example
signed Can store positive & negative signed int x = -10;
unsigned Only positive values unsigned int y = 10;
short Smaller size short int a = 100;
long Larger size long int b = 100000;

3. Constants in C

Constants are fixed values that cannot change during program execution.

const int MAX = 100;
#define PI 3.1416

4. Type Casting in C

Convert a variable from one type to another.

int x = 5;
float y = x;      // Implicit casting
float f = 5.9;
int a = (int)f;   // Explicit casting

5. Example Programs

#include <stdio.h>
int main() {
    int age = 20;
    float height = 5.9;
    char grade = 'A';
    printf("Age: %d, Height: %.1f, Grade: %c\n", age, height, grade);
    return 0;
}

Best Practices:
- Use meaningful variable names
- Initialize variables before use
- Use const for constants
- Avoid magic numbers; use macros/constants
